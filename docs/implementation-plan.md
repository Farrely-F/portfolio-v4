# Portfolio Website — Implementation Plan

> **Role:** Senior UX & Front-End Engineer  
> **Source Data:** `docs/resume.md`  
> **Tech Stack:** Next.js 16 (App Router), React 19, Strict TypeScript, Tailwind CSS v4, shadcn/ui (radix-vega), Framer Motion

---

## 1. Overview & Design Philosophy

Build a **single-page portfolio** (SPA-style scroll experience) that acts as a living résumé. The site must feel premium, performant, and accessible — reflecting the seniority of a Front-End Engineer with 3+ years of production experience.

### Core Principles
- **Content-first:** Every animation and visual effect must serve the content from `docs/resume.md`.
- **Progressive enhancement:** The site is fully readable with JS disabled; motion layers on top.
- **Performance budget:** Keep First Contentful Paint (FCP) under 1.2s and Total Blocking Time (TBT) under 200ms.
- **Accessibility:** WCAG 2.1 AA compliant — respect `prefers-reduced-motion`, maintain focus rings, semantic landmarks.

- **Progress indicator:** Thin animated line at the top showing scroll depth (optional P2).

## 5. Component Architecture

### Server vs Client Boundaries
Following Next.js App Router best practices (referencing `node_modules/next/dist/docs/01-app/01-getting-started/05-server-and-client-components.md`):

| Layer | Type | Responsibility |
|---|---|---|
| **Page / Layout** | Server Component | Data ingestion, metadata, global structure |
| **Section wrappers** | Server Component | Static layout, pass data down |
| **Motion wrappers** | Client Component (`"use client"`) | Framer Motion `motion.div`, scroll triggers, hover |
| **Interactive UI** | Client Component | Mobile menu, theme toggle, tooltips |
| **shadcn primitives** | Client Component | Button, Badge, Card, Dialog (already `"use client"`) |
| **Data utilities** | Server-only module | Raw résumé data parsing / static JSON |

### Key Patterns
1. **Data colocation:** Store résumé data as a typed TypeScript object in `src/lib/data.ts` (Server Component safe). Import into page sections.
2. **Animation isolation:** Create reusable `FadeIn`, `StaggerContainer`, and `TextReveal` components in `src/components/motion/` so only those files carry `"use client"`.
3. **No context pollution:** Wrap third-party interactive libs (Framer Motion) in thin local wrappers so `"use client"` does not leak upstream.

## 6. Animation Strategy (Framer Motion)

### Global Rules
- Always gate animations behind `prefers-reduced-motion`.
- Use `transform` and `opacity` only (GPU-composited properties).
- Avoid layout thrashing — no width/height animations.

### Section Animations

| Effect | Component | Trigger | Easing |
|---|---|---|---|
| **Fade Up** | `FadeIn` | `whileInView` | `[0.22, 1, 0.36, 1]` (ease-out-expo) |
| **Stagger List** | `StaggerContainer` | `whileInView` | Children delay `0.05s` |
| **Text Reveal** | `TextReveal` | `whileInView` | Masked clip-path or y-translate |
| **Hover Lift** | `Card` | `whileHover` | `y: -4`, `scale: 1.01` |
| **Hero Entrance** | `HeroSection` | `initial` / `animate` | Staggered `0.1s` children |

### Specific Behaviors
- **Hero:** Name and tagline stagger in on mount. CTA buttons fade up last.
- **Projects:** Cards fade up in a staggered grid as user scrolls into view.
- **Tech Stack:** Tags animate in with a spring stagger.
- **Navigation:** Desktop links have an underline scale-X hover effect. Mobile menu slides in from right.

## 8. Phase-by-Phase Implementation

### Phase 0: Foundation & Tooling (1–2 hours)
1. Install dependencies: `framer-motion`, `next-themes`.
2. Add shadcn components: `npx shadcn add badge card sheet`.
3. Configure fonts in `src/app/layout.tsx` using `next/font/google`.
4. Set up theme tokens in `src/app/globals.css` (Tailwind v4 `@theme` block).
5. Create `src/lib/data.ts` — type and populate all résumé data.
6. Update `next.config.ts` with static export or image config if needed.

### Phase 1: Static Shell (2–3 hours)
1. Build `src/components/sections/` as pure static components with zero animation.
2. Compose them in `src/app/page.tsx`.
3. Build `Navbar` and `Footer`.
4. Ensure semantic HTML (`<header>`, `<main>`, `<section>`, `<footer>`, `<nav>`).
5. Validate responsive layout at mobile, tablet, and desktop breakpoints.
6. Run `npm run build` to verify zero TypeScript errors.

### Phase 2: Motion Layer (2–3 hours)
1. Implement `FadeIn`, `StaggerContainer`, `TextReveal` in `src/components/motion/`.
2. Wrap each section with appropriate motion components.
3. Add `whileInView` triggers with `once: true` to prevent re-triggering on scroll back.
4. Add `prefers-reduced-motion` guard inside motion wrappers.
5. Test on low-powered devices; disable heavy effects if needed.

### Phase 3: Polish & Interactions (1–2 hours)
1. Add hover states to project cards, nav links, and buttons.
2. Implement mobile menu using shadcn `Sheet`.
3. Add smooth scroll behavior (`scroll-behavior: smooth` on `html`).
4. Implement theme toggle (dark / light) using `next-themes`.
5. Add `aria-label`, `aria-expanded`, and focus rings for accessibility.

### Phase 4: Performance & SEO (1 hour)
1. Generate OG image (`opengraph-image.tsx`) and metadata in `layout.tsx`.
2. Add `sitemap.ts` and `robots.ts`.
3. Run Lighthouse audit; target 95+ Performance, 100 Accessibility.
4. Verify no hydration errors (common with `next-themes`).
5. Optional: Add `@vercel/analytics`.

## 12. Next.js v16 Specific Considerations

> Per `AGENTS.md`, this Next.js version has breaking changes. Key conventions observed from `node_modules/next/dist/docs/`:

1. **App Router is primary.** File system routing uses `app/page.tsx` and `app/layout.tsx`.
2. **Server Components by default.** Only mark files `"use client"` when state, effects, or browser APIs are required.
3. **Params are Promises.** Dynamic route params (`params`) and `searchParams` are async — `await` them before use.
4. **Tailwind v4 integration.** Use `@import 'tailwindcss'` in global CSS and `@theme` for custom tokens. No `tailwind.config.js` required for most cases.
5. **CSS ordering.** Import global/Tailwind styles in the root layout only. Component-level styles should use Tailwind utilities or CSS Modules.
6. **Type helpers.** `PageProps<'/route'>` and `LayoutProps<'/route'>` are globally available after running `next dev` or `next build`.
